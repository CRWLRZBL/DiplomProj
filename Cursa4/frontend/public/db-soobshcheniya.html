Задействованные таблицы БД (модуль «Сообщения»)

Описание таблиц SQL Server, которые использует функциональность внутренних сообщений: переписка клиента с салоном и чат между сотрудниками (администратор / менеджер).

Скрипт создания: backend/CourseProjectAPI/Scripts/chat_tables.sql. Связанные сущности пользователей уже есть в основной схеме приложения (Users, Roles, UserProfiles).

Схема связей (упрощённо)

┌─────────────────┐         ┌──────────────────────┐
│     Users       │         │  ChatConversations   │
│  (UserID PK)    │◄────────│  ClientUserId (FK)   │
└────────┬────────┘         │  StaffKeyUser1 (FK)  │
         │                  │  StaffKeyUser2 (FK)  │
         │                  └──────────┬───────────┘
         │                             │
         │                  ┌──────────▼───────────┐
         └──────────────────►  ChatMessages        │
                            │  SenderUserId (FK)  │
                            │  ChatConversationId │
                            └─────────────────────┘

Таблица ChatConversations

Один ряд — одна «тема» переписки: либо поддержка клиента, либо прямой чат двух сотрудников.

Колонка Тип Назначение
ChatConversationId INT, IDENTITY, PK Идентификатор переписки
ConversationType TINYINT 0 — клиент ↔ салон (ClientSupport); 1 — два сотрудника (StaffDirect)
ClientUserId INT NULL, FK → Users(UserID) Для типа 0: клиент-владелец треда; уникален среди таких строк
StaffKeyUser1 INT NULL, FK → Users(UserID) Для типа 1: меньший из двух UserID участников
StaffKeyUser2 INT NULL, FK → Users(UserID) Для типа 1: больший из двух UserID участников
CreatedAt DATETIME2 Время создания
UpdatedAt DATETIME2 Обновляется при новом сообщении

Уникальные индексы: по ClientUserId (где NOT NULL); по паре (StaffKeyUser1, StaffKeyUser2) для типа 1.

Таблица ChatMessages

Отдельные сообщения внутри переписки.

Колонка Тип Назначение
ChatMessageId INT, IDENTITY, PK Идентификатор сообщения
ChatConversationId INT, FK → ChatConversations Принадлежность к треду; при удалении треда сообщения удаляются каскадом
SenderUserId INT, FK → Users(UserID) Кто отправил
Body NVARCHAR(2000) Текст сообщения
CreatedAt DATETIME2 Время отправки

Связанные таблицы основного приложения

Модуль сообщений не создаёт эти таблицы, но ссылается на них:

  • Users — отправитель, клиент в треде, участники staff-чата
  • Roles — роль пользователя (Client / Manager / Admin); логика API различает клиента и сотрудника
  • UserProfiles — имя и фамилия для отображения в интерфейсе (подгружается API)
